About this project

This static application demonstrates a lightweight exploration of multiple podcasts. It is entirely client‑side: it fetches a single dataset.json file and renders the contents without any backend API. You can host the directory containing index.html, dataset.json and docs.html on any static file server, and the site will continue to function.

Features

  • A list of all included podcasts, selectable in the sidebar.
  • A list of episodes for the selected podcast, including title, publication date and description.
  • Dark‑mode support. The interface automatically picks up your system preference, and you can override it using the toggle in the sidebar. Your choice is stored in localStorage.
  • A simple navigation between the home view and this documentation.

Extensibility

This demo intentionally keeps the data model minimal: each episode record contains a kind classification (special, meta or regular) and a cluster_id based on its publication decade. If you wish to add richer information—such as historical time spans, places, people or topics—you can extend the JSON structure and update index.html accordingly. The modular design of the user interface makes it straightforward to incorporate additional panels or visualisations.

Building the dataset

The dataset is built using the accompanying Python script build_dataset.py, which accepts multiple RSS files and outputs a consolidated JSON file. For example: 

python build_dataset.py --rss mp3.rss --rss feed --rss esh.xml --rss rest-gesch.xml --out dataset.json

You can run this command in an environment with Python 3.9+ to regenerate dataset.json whenever the feeds change.
